# The Vibe Coding Loop

## Plan van aanpak:
Schrijf een MD vanuit PBI beschrijving. Of laat copilot dit doen. Zorg ervoor dat verschillende fases in deze MD staan. Per fase zou je een PR kunnen opleveren.

## In een fase voer deze stappen uit:
* 1-3 plan mode
* 4-6 reviewen en itereren
* 7 bonus

## Werk in fases:
* Fase 1: Domain Layer — Pure Business Logic
* Fase 2: Service & API Layer — Integration & In-Memory Component Tests
* Fase 3: Database Schema — Add Table to Database.sqlproj
* Fase 4: Frontend — UI Components

## Stappen per fase:
[https://itrevolution.com/articles/the-vibe-coding-loop/](https://itrevolution.com/articles/the-vibe-coding-loop/)

1.	Frame your objective
Geef je AI-samenwerkingspartner een duidelijke en beknopte beschrijving van het resultaat dat je wilt bereiken. Wees specifiek over hoe succes eruitziet en waarom je dit bouwt.
	
2.	Decompose the tasks
Verdeel wat je wilt doen in duidelijke, haalbare stappen. Over het algemeen geldt: hoe kleiner de stappen, hoe groter de kans dat AI slaagt. Zelfs naarmate AI krachtiger wordt, blijven kleine stappen een goed idee. Aarzel niet om de grote taken verder te laten opdelen (bijvoorbeeld: “Dit is wat ik wil bereiken. Stel een plan voor.”).

3.	Start the conversation
Vraag AI om een plan te maken om je doel te bereiken, of geef instructies om het op gang te helpen, zoals je in het vorige hoofdstuk hebt geoefend.
Gebruik:  ‘PLAN’

4.	Review with care
De oplossing die AI aandraagt kan correct lijken, maar totdat je een basis hebt om erop te vertrouwen, moet je deze grondig nakijken. Review de changes / staged code handmatig.

5.	Test and verify
Jij bent verantwoordelijk voor de kwaliteit van de code, of je die nu zelf hebt geschreven of AI dat heeft gedaan. Dit werkt het best wanneer je je tests en verwachtingen opstelt vóórdat je de code genereert—voorstanders van test-driven development (TDD) zullen dit toejuichen. Faal snel, herstel snel, en vraag AI om je te helpen subtiele fouten op te sporen die anders onopgemerkt blijven.

6.	Refine and iterate
Blijf itereren totdat je je doel hebt bereikt.


### (BONUS) zodra je wat ervaring hebt met deze vibe-codingcyclus, is er nog een belangrijke stap om toe te voegen:

7.	Automate yourself
Begin delen van je workflow te automatiseren. Elke vorm van frictie brengt grote opportuniteitskosten met zich mee. En elke keer dat je tijd besteedt aan typen, kopiëren/plakken of handmatig slepen, vertraag je je vibe-codingcyclus. Als je iets handmatig doet, is dat een kost die je telkens opnieuw betaalt wanneer je probeert te vibe-coden.
